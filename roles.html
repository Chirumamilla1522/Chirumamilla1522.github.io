This application is meant for Onsite Employees, HR, and Admin whose roles have been defined below:

Role of employee:

  • The user will login to the system and his image will be captured along with his GPS location and all the information will be sent to the admin where admin will view image and GPS location.
  • Once logged in, the GPS location of the employee will be tracked auto- matically by the system and sent to the admin after every 5 minutes. For tracking GPS location, you may opt for using available GPS libraries in your implementation.
  • When user logout the system again the image will be captured as well as GPS location will be sent to the admin.ÃŸ

    • Role of Admin:

  • The role of the admin is to add new employee by entering his personal details and provides the employee with employeeID and password to the user so that he can access the applicati
  • Admin can view the GPS location of the employee by entering Employee Identity Number as well as Date. (Use Google maps to display the various locations of an employee)
  • Admin can check the salary of the particular employee by entering date and employee ID.
  • Admin can view latitude and longitude of the GPS location sent by the employee.
  • Admin can change the password of the employee.

    • Role of HR:

  • When the HR login to the system he can check the GPS location of the employee by entering employee ID and date.
  • HR can check salary of the particular employee by entering employee ID and date.
